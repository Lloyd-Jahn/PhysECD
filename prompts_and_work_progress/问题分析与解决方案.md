# PhysECD 训练问题分析与解决方案

## 问题发现过程

### 初始问题
训练时发现四个指标的 loss 数量级相差很大，R loss 高达 4000-5000 且无法收敛。

## 深度分析

### 问题1：数据集公式错误
**发现**：原始代码使用 `R = 471.44 × (μ · m)`，但数据集实际符合 `R = 6414.135 × (μ · m) / E`

**验证结果**：
- 旧公式相对误差：~50%
- 新公式相对误差：~2.9%

**解决**：修改 `aggregation.py` 使用正确公式

### 问题2：E_pred 无物理约束
**发现**：激发能预测头输出无约束，导致 E_pred 可能为负数或接近0

**后果**：
- E_pred ∈ [-0.43, 0.47] eV（应该是 3-9 eV）
- R = (μ·m) / E_pred 会除以负数或接近0的数
- 数值不稳定，梯度爆炸

**解决**：在 `heads.py` 中添加 Softplus 激活函数
```python
E_pred = F.softplus(E_raw) + 3.0  # 保证 E_pred >= 3.0 eV
```

### 问题3：Loss 尺度严重不平衡（核心问题）
**数据统计**：
- E: mean=6.44, std=1.12
- μ: mean=0.069, std=0.109
- m: mean=0.354, std=0.560
- R: mean=30.23, std=62.06

**初始 MSE（预测为0时）**：
- MSE(E) ≈ 45
- MSE(μ) ≈ 0.012
- MSE(m) ≈ 0.314
- MSE(R) ≈ 3851

**加权后贡献**（lambda_E=1, lambda_mu=10, lambda_m=50, lambda_R=1）：
- E: 1.0 × 45 = 45
- μ: 10.0 × 0.012 = 0.12
- m: 50.0 × 0.314 = 15.7
- R: 1.0 × 3851 = **3851**

**结论**：R loss 占总 loss 的 98%，完全主导训练，导致模型忽略 μ 和 m 的学习。

## 最终解决方案

### 两阶段训练策略

**阶段1（Epoch 1-15）：Warmup Phase**
- `lambda_R = 0.0`（不监督 R）
- 只训练 E、μ、m
- 让模型先学好基础的偶极矩

**阶段2（Epoch 16-25）：Fine-tune Phase**
- `lambda_R = 0.1`（加入 R 监督）
- 在 μ 和 m 已经学好的基础上优化 R
- 此时 R 的误差会小很多，不会主导训练

### 代码修改总结

1. **aggregation.py**
   - 使用正确公式：`R = 6414.135 × (μ·m) / E`
   - 添加数值稳定性保护：`E_pred_safe = clamp(E_pred, min=1.0)`

2. **heads.py**
   - 添加物理约束：`E_pred = softplus(E_raw) + 3.0`
   - 保证 E_pred >= 3.0 eV

3. **loss.py**
   - 移除旧的单位转换系数
   - R_pred 已经在正确单位下

4. **04_train.py**
   - 实现两阶段训练
   - 动态调整 lambda_R

## 预期效果

### 阶段1（Warmup）
- loss_E 应该降到 ~0.5
- loss_mu 应该降到 ~0.01
- loss_m 应该降到 ~0.1-0.3
- loss_R 不参与训练

### 阶段2（Fine-tune）
- 在 μ 和 m 已经准确的基础上
- loss_R 应该能降到 ~10-100（相比初始的 3850，下降 40-400 倍）
- 最终 R 的相对误差应该在 10-20% 左右

## 关键洞察

1. **物理公式必须正确**：错误的公式导致数据不一致，模型无法学习
2. **物理约束必须满足**：E > 0 是硬约束，必须在模型中体现
3. **Loss 平衡至关重要**：不能让某一项 loss 主导训练
4. **分阶段训练更有效**：先学基础量，再学复合量
5. **R 应该通过 μ 和 m 间接优化**：直接优化 R 会导致梯度冲突
